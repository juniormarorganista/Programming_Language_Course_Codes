# Contribuindo

## Fluxo obrigatório

1. crie uma issue;
2. crie uma branch curta a partir de `main`;
3. faça commits pequenos e claros;
4. abra um PR;
5. faça merge;
6. delete a branch.

## Convenção de branches

- `feat/...` para conteúdo novo;
- `docs/...` para documentação;
- `fix/...` para correções;
- `chore/...` para manutenção;
- `final/...` para consolidação final de blocos.

## Convenção de commits

- `feat: ...`
- `docs: ...`
- `fix: ...`
- `refactor: ...`
- `test: ...`
- `chore: ...`

## O que evitar

- commit gigante e vago;
- branch longa demais;
- misturar refatoração, conteúdo e documentação no mesmo commit;
- subir binário compilado sem necessidade.
