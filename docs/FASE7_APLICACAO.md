# Aplicação da fase 7

Esta fase é a última camada de acabamento estrutural do repositório.

## Entregas da fase

- scripts de auditoria de cabeçalhos;
- script de aplicação controlada de cabeçalhos genéricos;
- script de reconstrução do índice de READMEs e módulos;
- validação PowerShell para Windows;
- template de issue para limpeza gradual do acervo legado.

## Fluxo recomendado de uso

1. Rode a validação da fase.
2. Gere o relatório de auditoria.
3. Faça um commit só com documentos e scripts.
4. Aplique cabeçalhos em um grupo pequeno de extensões.
5. Revise manualmente o resultado.
6. Faça um segundo commit só com cabeçalhos aplicados.

## Separação boa de commits

- commit 1: ferramentas e documentos;
- commit 2: cabeçalhos em Python e SQL;
- commit 3: cabeçalhos em C, C++, Java e PHP;
- commit 4: revisão manual dos casos científicos e especiais.
