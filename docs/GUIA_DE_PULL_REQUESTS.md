# Guia de Pull Requests e Releases

## Pull request bom
Um PR deste repositório deve responder:
- o que foi criado ou alterado;
- por que isso melhora a trilha;
- como testar;
- quais arquivos principais foram tocados.

## Modelo de título
- `feat: add java oop starter files`
- `docs: improve frontend learning roadmap`
- `refactor: reorganize php mini projects`
- `chore: validate repository structure scripts`

## Antes de abrir PR
- rode os scripts de validação da fase;
- confira o `git status`;
- revise README e comandos;
- garanta que não subiu binário sem querer.
