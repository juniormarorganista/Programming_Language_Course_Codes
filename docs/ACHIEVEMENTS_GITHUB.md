# GitHub, portfólio e Achievements

## Verdade prática

O melhor jeito de fortalecer o perfil no GitHub não é sair caçando badge como se fosse figurinha rara.
É produzir um histórico que pareça trabalho sério.

## O que ajuda de verdade

- commits frequentes e organizados;
- branches curtas;
- Pull Requests claros;
- issues bem descritas;
- releases marcando marcos reais;
- documentação boa;
- evolução visível dos módulos.

## Fluxo recomendado

1. abrir issue;
2. criar branch;
3. implementar;
4. testar;
5. commitar;
6. abrir PR;
7. revisar;
8. mergear;
9. criar release quando fizer sentido.

## Exemplos de issues úteis

- `[MODULO] PHP - Arrays`
- `[MODULO] Java - Coleções`
- `[MELHORIA] Reescrever README principal`
- `[MELHORIA] Separar SQL em schema/inserts/queries`

## Como isso ajuda o perfil

Esse fluxo:
- mostra consistência;
- melhora histórico público;
- deixa o projeto navegável;
- prova organização;
- demonstra que você sabe evoluir software, não só empilhar arquivo.

## O que evitar

- commits gigantes sem contexto;
- PR com cinquenta mudanças desconexas;
- arquivos sem instrução de uso;
- código binário versionado à toa;
- módulos sem README.

## Meta saudável

Em vez de “quero achievement”, trabalhe com metas assim:
- 1 módulo por semana;
- 1 PR por módulo;
- 1 release a cada fase relevante;
- 1 melhoria documental a cada sprint.

É menos glamouroso e muito mais útil.
