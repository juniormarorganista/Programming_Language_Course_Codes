# GitHub e evolução do perfil

## Ideia central

O objetivo aqui não é caçar badge como quem junta figurinha no escuro. O objetivo é usar o GitHub direito.

## O que fazer

- abrir issues por módulo;
- usar branches curtas;
- escrever commits claros;
- abrir pull requests bem descritos;
- usar releases para marcos importantes;
- manter README e documentação atualizados.

## Exemplo de ciclo saudável

1. criar uma issue: `Módulo Java Básico`;
2. criar uma branch: `feat/java-basics`;
3. adicionar arquivos e README;
4. commitar;
5. subir a branch;
6. abrir PR;
7. revisar;
8. fazer merge;
9. marcar release quando concluir uma fase.

## O que evitar

- commits gigantes com cinquenta assuntos;
- branch eterna;
- arquivo solto sem documentação;
- README prometendo o que o repositório não entrega.

## Resultado esperado

Mesmo sem inventar moda, esse fluxo fortalece:

- histórico do projeto;
- clareza para quem visita o perfil;
- disciplina de estudo;
- sinal profissional no GitHub.
