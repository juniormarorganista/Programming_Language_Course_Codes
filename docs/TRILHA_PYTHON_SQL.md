# Trilha Python + SQL

Este documento organiza a próxima etapa prática do repositório.

## Objetivo

Transformar Python e SQL em duas trilhas realmente utilizáveis para estudo contínuo, revisão e produção de commits consistentes no GitHub.

## Python

### Módulo 1 — Fundamentos
- saída padrão
- variáveis
- tipos básicos
- condicionais
- laços
- funções
- listas, tuplas, dicionários e conjuntos
- strings
- arquivos
- exceções

### Módulo 2 — Exercícios guiados
- jogo de adivinhação
- analisador de frases
- agenda simples

### Módulo 3 — Projetos pequenos
- boletim de notas em linha de comando
- controle simples de despesas

### Módulo 4 — Ponte para análise de dados
- estatísticas com biblioteca padrão
- leitura e análise de CSV

## SQL

### Módulo 1 — Base relacional
- criação de tabelas
- chaves primárias e estrangeiras
- inserções
- consultas simples

### Módulo 2 — Joins
- `INNER JOIN`
- `LEFT JOIN`

### Módulo 3 — Agregação
- `COUNT`
- `SUM`
- `AVG`
- `GROUP BY`
- `HAVING`

### Módulo 4 — Subconsultas
- filtros com subconsulta
- comparação com média global

### Módulo 5 — Views
- visão reutilizável para relatórios

## Como estudar

1. rode um exemplo;
2. altere manualmente o código;
3. registre o que mudou no commit;
4. abra uma branch por módulo;
5. suba PR curto e descritivo.

## Meta de GitHub saudável

- 1 issue por módulo
- 1 branch por módulo
- 1 PR por módulo
- 1 release por bloco fechado
