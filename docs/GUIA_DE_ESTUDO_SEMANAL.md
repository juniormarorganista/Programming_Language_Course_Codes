# Guia de Estudo Semanal

## Estrutura sugerida

### Segunda
- leitura do README da linguagem da semana
- revisão dos exemplos básicos

### Terça
- execução dos exemplos
- anotações do que travou

### Quarta
- exercícios guiados
- correção de bugs pequenos

### Quinta
- mini projeto ou refatoração
- melhoria de documentação

### Sexta
- commit limpo
- push da branch
- abertura de PR

### Sábado
- revisão geral
- issues para próxima semana

### Domingo
- descanso ou leitura leve

## Sessão mínima produtiva
Mesmo em dia ruim, faça pelo menos:
- ler 1 arquivo;
- rodar 1 exemplo;
- fazer 1 commit.

É pouco, mas evita o cemitério de planos perfeitos.
