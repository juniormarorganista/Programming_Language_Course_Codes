# Revisão transversal do repositório

Este documento serve como checklist para manter o repositório coerente à medida que novas linguagens e módulos são adicionados.

## 1. Estrutura

Cada linguagem deve ter:
- `README.md`
- subpastas com progressão didática
- ao menos um mini projeto
- convenção de nomes consistente

## 2. Cabeçalho dos exemplos

Todo arquivo novo deve começar, em comentário, com:
- Objetivo
- Como executar
- Conceitos treinados
- Próximo exercício

## 3. Versionamento

Fluxo recomendado:
1. criar uma issue por módulo
2. criar uma branch curta
3. commitar com mensagem objetiva
4. abrir PR
5. registrar release em marcos importantes

## 4. Portfólio

Ao selecionar projetos para destacar no GitHub, priorize:
- clareza do README
- instruções de execução
- objetivos explícitos
- histórico de evolução

## 5. Achievements

Não persiga badges como se fossem figurinha rara em álbum de infância tardia. O caminho consistente é:
- commits frequentes
- PRs bem feitos
- discussões quando fizer sentido
- issues úteis
- releases com versão

## 6. Próximas frentes

Depois desta fase, os candidatos naturais são:
- reforçar `frontend/react`
- expandir `java` e `php`
- criar exercícios comparativos entre linguagens
- adicionar CI simples para verificar estrutura e arquivos essenciais
