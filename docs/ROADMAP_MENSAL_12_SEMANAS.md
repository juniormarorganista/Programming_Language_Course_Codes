# Roadmap Mensal de 12 Semanas

## Meta
Conduzir o repositório como trilha de estudo e portfólio por 12 semanas.

## Semanas 1 a 4 — Base forte

### Semana 1
- Revisar estrutura do repositório
- Validar scripts locais
- Estudar Python basics
- Criar 1 PR pequeno

### Semana 2
- Consolidar Python projects
- Estudar SQL basics e joins
- Criar release `v0.1-study-foundation`

### Semana 3
- Estudar C basics
- Revisar `.gitignore`, READMEs e headers
- Abrir issues por módulo

### Semana 4
- Estudar C++ basics e mini projeto
- Revisão geral da documentação
- Fechar milestone “Base forte”

## Semanas 5 a 8 — Expansão prática

### Semana 5
- Estudar Java basics
- Compilar exemplos localmente
- Abrir PR do módulo Java

### Semana 6
- Estudar Java OOP e files
- Criar mini projeto Java

### Semana 7
- Estudar PHP basics e forms
- Rodar servidor local com `php -S`

### Semana 8
- Estudar Frontend: HTML/CSS + JavaScript
- Criar página estática e página com DOM
- Release `v0.2-web-and-oop`

## Semanas 9 a 12 — Trilhas matemáticas e acabamento

### Semana 9
- Estudar MATLAB/Octave basics
- Revisar numerical methods

### Semana 10
- Estudar R data e statistics
- Gerar gráficos e relatórios simples

### Semana 11
- Estudar Mathematica basics e symbolic
- Revisão transversal de convenções

### Semana 12
- Fechar checklist final
- Limpar pendências do GitHub
- Criar release `v1.0-learning-repo-foundation`

## Regra operacional
Cada semana deve terminar com:
- 1 branch fechada;
- 1 PR aberto ou mergeado;
- 1 README atualizado;
- 1 commit que melhore documentação ou estrutura.
