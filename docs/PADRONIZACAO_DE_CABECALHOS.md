# Padronização de cabeçalhos para exemplos antigos

O padrão mínimo adotado nesta fase é:

- **Objetivo:** o que o arquivo treina ou demonstra.
- **Como executar:** comando mínimo ou observação de execução.
- **Conceitos treinados:** tópicos principais.
- **Próximo exercício:** passo sugerido de continuação.

## Exemplo em Python

```python
# Objetivo: demonstrar leitura de entrada e uso de condicionais.
# Como executar: python nome_do_arquivo.py
# Conceitos treinados: input, if, elif, else.
# Próximo exercício: transformar este exemplo em função reutilizável.
```

## Exemplo em C / C++ / Java / JavaScript / TypeScript / PHP / SQL / R / MATLAB / Fortran

O mesmo bloco é aplicado com prefixo de comentário adequado à linguagem.

## Exemplo em HTML

```html
<!--
Objetivo: demonstrar estrutura básica de página e estilização simples.
Como executar: abrir este arquivo no navegador.
Conceitos treinados: tags semânticas, CSS básico.
Próximo exercício: adicionar interação com JavaScript.
-->
```

## Exemplo em Wolfram Language

```wolfram
(*
Objetivo: demonstrar manipulação algébrica simbólica.
Como executar: wolframscript -file arquivo.wls
Conceitos treinados: expressões simbólicas e simplificação.
Próximo exercício: adicionar um gráfico do resultado.
*)
```

## Regra prática

Se não for possível inferir o objetivo com segurança, use um cabeçalho genérico e marque o arquivo para revisão manual posterior. Melhor um rótulo honesto do que uma descrição inventada.
